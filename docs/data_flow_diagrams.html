Resume Screening and Career Guidance System

1. Introduction to Data Flow Diagrams

Data Flow Diagrams (DFDs) are graphical representations that illustrate how data moves through an information system. They show the flow of data between external entities, processes, and data stores. DFDs help in understanding the system's functionality without getting into technical implementation details.

4.1 Sub-Process Description

Process ID Process Name Description Technology Used
2.1 Text Extraction Extracts raw text content from PDF and DOCX files using layout-aware parsing. Handles tables separately. pdfplumber, python-docx
2.2 Text Cleaning Removes excessive whitespace, normalizes line breaks while preserving document structure. Python regex
2.3 Contact Extraction Identifies email addresses, phone numbers (multiple formats), GitHub/LinkedIn links. Regular expressions
2.4 Skill Identification Matches text against skills database using word boundaries. Normalizes skill variations to canonical names. Pattern matching, word boundaries
2.5 Experience Detection Extracts years of experience from various text patterns like "5+ years experience" or date ranges. Regular expressions
2.6 Entity Recognition Uses NLP to identify person names, organizations, educational institutions, and geographic locations. spaCy Named Entity Recognition

4.2 Data Flow Details

Data Flow From To Description
PDF/DOCX File D1: Resume Database 2.1: Text Extraction Binary file content for parsing
Raw Text 2.1: Text Extraction 2.2: Text Cleaning Unprocessed text with layout preserved
Table Data 2.1: Text Extraction 2.2: Text Cleaning Content from tables in the document
Clean Text 2.2: Text Cleaning 2.3, 2.4, 2.5, 2.6 Normalized text for analysis
Skills List D2: Skills Database 2.4: Skill Identification Master list of recognized skills
Skill Variations D2: Skills Database 2.4: Skill Identification Alternative names for skills (js → JavaScript)
Matched Skills 2.4: Skill Identification D5, Process 3.0 List of identified skills
Complete Profile D5: Extracted Data Process 3.0, 4.0 Aggregated candidate information
Note: Process 2.4 (Skill Identification) uses a canonical mapping system that normalizes skill variations. For example, "js", "javascript", "node", "nodejs" are all mapped to their canonical form "JavaScript" and "Node.js" respectively. This ensures consistent skill matching across different resume formats.

5.2 Key System Characteristics

  • Two User Types: HR Users and Candidates interact with the system differently
  • Four Core Processes: Upload, Analysis, Scoring, and Guidance
  • Four Data Stores: Resume, Skills, Job Roles, and Score Records
  • Bidirectional Data Flow: Users both send and receive data
  • Modular Processing: Analysis is decomposed into specialized sub-processes

5.3 Data Flow Principles Applied

  • Each process transforms input data into output data
  • Data stores provide persistent storage between processes
  • External entities exist outside the system boundary
  • All data flows are labeled with descriptive names
  • No data flow connects two data stores directly (must pass through a process)
  • No data flow connects two external entities directly
