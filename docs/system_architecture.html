Resume Screening and Career Guidance System

1. Introduction

This document presents the system architecture and data flow diagrams for the Resume Screening and Career Guidance System. The system is designed to automate the resume screening process for HR professionals and provide career guidance to job seekers through skill gap analysis and personalized roadmaps.

The architecture follows a three-tier design pattern with clear separation between the presentation layer (React.js), application layer (Node.js/Express), and processing layer (Python/FastAPI). This modular approach ensures maintainability, scalability, and ease of future enhancements.

2.1 Architecture Layers

Presentation Layer (Frontend)

The frontend is built using React.js with Vite as the build tool. It provides responsive user interfaces for both HR professionals and candidates. Key components include dashboard views, resume upload functionality, filtering panels, and result visualization.

  • Technology: React.js, React Router, Tailwind CSS
  • Communication: RESTful API calls via Axios
  • Features: Single Page Application (SPA), Responsive Design
Application Layer (Backend)

The backend server is built using Node.js with Express.js framework. It handles HTTP requests, manages file uploads using Multer, implements business logic, and communicates with the processing service.

  • Technology: Node.js, Express.js, Multer
  • API Design: RESTful architecture
  • Features: Authentication, File Management, Request Routing
Processing Layer (Python Service)

The processing service is built using FastAPI (Python) and handles all resume analysis operations. It performs text extraction, skill identification, and similarity scoring using NLP techniques.

  • Technology: Python, FastAPI, spaCy, sentence-transformers
  • Functions: Resume Parsing, Skill Extraction, Similarity Calculation
  • Features: High Performance, Concurrent Processing

5.1 Scoring Formula

The final match score is calculated using a weighted combination of three metrics:

Final Score = (0.45 × Skill Match) + (0.40 × Semantic Similarity) + (0.15 × Keyword Similarity)
Component Weight Method Purpose
Skill Match Ratio 45% Set intersection of candidate skills and required skills Direct measurement of skill coverage
Semantic Similarity 40% BERT embeddings with cosine similarity Captures meaning and context alignment
Keyword Similarity 15% TF-IDF vectors with cosine similarity Ensures important terms are present

9. Conclusion

The Resume Screening and Career Guidance System architecture is designed with modularity, scalability, and maintainability in mind. The three-tier architecture ensures clear separation of concerns:

  • Presentation Layer: Handles user interaction with responsive React interfaces
  • Application Layer: Manages business logic, routing, and file handling
  • Processing Layer: Performs computationally intensive NLP operations

The Data Flow Diagrams presented in this document illustrate the systematic flow of information from user input to final output, ensuring transparency and traceability of data processing.
